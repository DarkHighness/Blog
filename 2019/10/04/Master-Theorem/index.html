主定理 Master Theorem

对于采用分治策略的算法, 其递归方程一般可用主定理口算求解

内容

假设有递推关系式 T(n)=aT(nb)+Θ(n), a≥1,b>1T(n) = aT(\dfrac{n}{b}) + \Theta(n), \space a \ge 1, b > 1T(n)=aT(bn​)+Θ(n), a≥1,b>1 其中,

  • nnn 为问题规模
  • aaa 为递归子问题的数量
  • nb\dfrac{n}{b}bn​ 为每个子问题的规模 假设每个子问题的规模基本一样
  • f(n)f(n)f(n) 为递归以外进行的计算

则该算法的渐进时间复杂度 O\OmicronO 可分为以下三种情形讨论

  • 情形1:

    如果 ∃ ϵ>0 (ϵ为一常数)\exists \space \epsilon > 0 \space (\epsilon 为一常数)∃ ϵ>0 (ϵ为一常数), 有 f(n)=O(nlog⁡ba−ϵ) (多项式地小于)f(n) = \Omicron (n^{\log_{b}{a} - \epsilon}) \space (多项式地小于)f(n)=O(nlogb​a−ϵ) (多项式地小于)

    则 T(n)=Θ(nlog⁡ba)T(n) = \Theta (n^{\log_{b}{a}})T(n)=Θ(nlogb​a)

  • 情形2:

    如果∃ k≥0, (k为一常数)\exists \space k \ge 0, \space (k为一常数)∃ k≥0, (k为一常数), 有 f(n)=Θ(nlog⁡balog⁡kn)f(n) = \Theta (n^{\log_{b}{a}} \log^{k}n)f(n)=Θ(nlogb​alogkn)

    则 T(n)=Θ(nlog⁡balog⁡k+1n)T(n) = \Theta (n^{\log_{b}{a}} \log^{k + 1}n)T(n)=Θ(nlogb​alogk+1n)

  • 情形3:

    如果 ∃ ϵ>0 (ϵ为一常数)\exists \space \epsilon > 0 \space (\epsilon 为一常数)∃ ϵ>0 (ϵ为一常数), 有 f(n)=Ω(nlog⁡ba+ϵ) (多项式地大于)f(n) = \Omega(n^{\log_{b}{a} + \epsilon}) \space (多项式地大于)f(n)=Ω(nlogb​a+ϵ) (多项式地大于)

    同时 ∃ c<1 (c为一常数)\exists \space c < 1 \space (c为一常数)∃ c<1 (c为一常数) 以及充分大的 nnn 满足 af(nb)≤cf(n)af(\dfrac{n}{b}) \le cf(n)af(bn​)≤cf(n)

    则 T(n)=Θ(f(n))T(n) = \Theta (f(n))T(n)=Θ(f(n))

实际应用

在实际使用中我们并不需要记得那么复杂…

大概就记住下面一串?

设递归方程为 T(n)=aT(nb)+c(nd) ,其中 a≥1 ,b>1则所求问题的复杂度 T(n)={Θ(ndlog⁡n) ,a=bdΘ(nd) ,a<bdΘ(nlog⁡ba) ,a>bd设递归方程为 \space T(n) = aT(\dfrac{n}{b}) + c(n^{d}) \space , 其中 \space a \ge 1 \space, b > 1 \\ 则所求问题的复杂度 \space T(n) = \begin{cases} \Theta (n^{d}\log{n}) \space , a = b ^ d \\ \\ \Theta (n^d) \space , a < b ^ d \\ \\ \Theta (n^{\log_{b}{a}}) \space , a > b ^ d \end{cases}设递归方程为 T(n)=aT(bn​)+c(nd) ,其中 a≥1 ,b>1则所求问题的复杂度 T(n)=⎩⎪⎪⎪⎪⎪⎪⎨⎪⎪⎪⎪⎪⎪⎧​Θ(ndlogn) ,a=bdΘ(nd) ,a<bdΘ(nlogb​a) ,a>bd​

举例

二分查找 Binary Search

每次问题规模减半, 则 a=1 ,b=2a = 1 \space , b = 2a=1 ,b=2

每次额外的计算为0, 则 d=0d = 0d=0

所以可得递推关系式 T(n)=T(n2)+c(1)T(n) = T(\dfrac{n}{2}) + c(1)T(n)=T(2n​)+c(1)

上式满足 a=bda = b ^ da=bd 即情形1

所以复杂度为 Θ(n0log⁡n)=Θ(log⁡n)\Theta (n^{0} \log{n}) = \Theta (\log{n})Θ(n0logn)=Θ(logn)

快速排序 Quick Sort

每次问题规模减半, 问题划分成两部分, 则 a=2,b=2a = 2, b = 2a=2,b=2

每次额外需要进行一次合并, 则 d=1d = 1d=1

所以可得递推关系式 T(n)=2T(n2)+c(n1)T(n) = 2T(\dfrac{n}{2}) + c(n^1)T(n)=2T(2n​)+c(n1)

上式满足 a=bda = b^da=bd 即情形1

所以复杂度为 Θ(n1log⁡n)=Θ(nlog⁡n)\Theta(n^1\log{n}) = \Theta(n\log{n})Θ(n1logn)=Θ(nlogn)

归并排序 Merge Sort

每次问题规模减半, 问题划分成两部分, 则 a=2,b=2a = 2, b = 2a=2,b=2

每次额外需要进行一次合并, 则 d=1d = 1d=1

所以可得递推关系式 T(n)=2T(n2)+c(n1)T(n) = 2T(\dfrac{n}{2}) + c(n^1)T(n)=2T(2n​)+c(n1)

上式满足 a=bda = b^da=bd 即情形1

所以复杂度为 Θ(n1log⁡n)=Θ(nlog⁡n)\Theta(n^1\log{n}) = \Theta(n\log{n})Θ(n1logn)=Θ(nlogn)

基数排序 Radix Sort

假设所取的基数为basebasebase则

每次问题规模变为1base\dfrac{1}{base}base1​,但子问题规模变为 basebasebase 倍, 则 a=base ,b=basea = base \space, b = basea=base ,b=base

每次需要分配一次桶为O(n)\Omicron(n)O(n), 则d=1d = 1d=1

所以可得递推关系式 T(n)=base∗T(nbase)+c(n1)T(n) = base * T(\dfrac{n}{base}) + c(n^1)T(n)=base∗T(basen​)+c(n1)

上式满足 a=bda = b^da=bd 即情形1

Radix sort complexity is O(wn)\Omicron(wn)O(wn) for n keys which are integers of word size w.
